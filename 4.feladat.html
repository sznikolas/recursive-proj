Hanoi tornyai - Tower of Hanoi


Feladat: Írjon programot, ami a legkevesebb lépéssel áthelyezi a lemezeket az A oszlopról a C oszlopra. Nagyobb lemezt kisebb lemezre tilos rakni. Lépésenként csak egy lemezt lehet mozgatni. A megvalósításhoz használjon rekurziót.



Hanoi tornyai

Hanoi tornyai

  A hanoi torony egy matematikai rejtvény. Három rúdból és különböző méretű lemezekből áll, amelyek bármelyik rúdra csúszhatnak. A rejtvény úgy kezdődik, hogy a lemezek méretük szerint növekvő sorrendben, rendezett halomban vannak az egyik rúdon, a legkisebb a tetején. A játék szabályai szerint az első rúdról az utolsóra kell átrakni a lemezeket minél kevesebb lépéssel úgy, hogy minden lépésben egy lemezt lehet áttenni, nagyobb lemez nem tehető kisebb lemezre, és ehhez összesen három rúd áll rendelkezésre. Úgy is ismert, mint Brahma tornyai, vagy világvége feladvány.


Hanoi tornyok megoldása C nyelvben
                
//C program a Hanoi tornyok megoldására

#include <stdio.h>

void towerOfHanoi(int, char, char, char);

int main()
{
    //Lemezek száma
    int num;

    printf("Adja meg a lemezek számát: ");
    scanf("%d", &num);
    printf("A Hanoi-toronyban szereplő lépések sorrendje a következő: \n");
    //A, B, C az oszlopok nevei
    towerOfHanoi(num, 'A', 'C', 'B');
    return 0;
}
void towerOfHanoi(int num, char frompeg, char topeg, char auxpeg)
{
    if (num == 1)
    {
        printf("\n 1. lemez áthelyezése %c oszlopról %c oszlopra", frompeg, topeg);
        return;
    }
    towerOfHanoi(num - 1, frompeg, auxpeg, topeg);
    printf("\n %d. lemez áthelyezése %c oszlopról %c oszlopra", num, frompeg, topeg);
    towerOfHanoi(num - 1, auxpeg, topeg, frompeg);
}


Forrás
  • tower(disk, source, dest, inter)
     IF disk = 1, THEN:
      move disk from source to dest
     ELSE:
      tower(disk - 1, source, inter, dest) // 1. lépés
      move disk from source to dest // 2. lépés
      tower(disk - 1, inter, dest, source) // 3. lépés
     END
  • Vegyük észre, hogy a pszeudokód egy base case-t ad hozzá: Ha a lemez = 1, akkor ez a legkisebb lemez. Ebben az esetben nem kell aggódnunk a többi lemez miatt, így egyszerűen közvetlenül áthelyezhetjük a legkisebb lemezt. A többi esetben a háromlépéses rekurzív eljárást követjük.


Az algoritmus leírása

Függvény leírása:

towerOfHanoi(disk, source, dest, inter) → Van egy rekurzív függvényünk, amely paraméterként azt a lemezt veszi fel, amely a legnagyobb lemez a toronyban és amit mozgatni szeretnénk. Ezenkívül a függvényünk három paramétert is elfogad, amelyek megadják, hogy a lemezt melyik rúdról kell áthelyezni (source), melyik rúdra kell áthelyezni (dest), és a 3. rudat, amelyet ideiglenesen használhatunk, hogy ez az áthelyezés megtörténhessen (inter).

 A legfelső szinten a teljes tornyot szeretnénk áthelyezni, tehát az 5-ös és kisebb lemezeket szeretnénk az A rúdról a C rúdra áthelyezni. Ezt három alapvető lépésre bonthatjuk.

 1.) A 4. és annál kisebb lemezek áthelyezése az A rúdról(forrás) a B rúdra(tartalék), a C rudat(cél) tartalékként használva. Hogyan csináljuk ezt? Rekurzív módon, ugyanezt az eljárást alkalmazva. Miután ezt befejeztük, az 5-nél kisebb lemezek mindegyike a B rúdra kerül.

 2.) Most, hogy az összes kisebb lemez a tartalék rúdon van, az 5-ös lemezt áthelyezhetjük az A rúdról(forrás) a C rúdra(cél).

 3.) Végül a 4-es és a kisebb lemezeket a B rúdról(tartalék) a C rúdra(cél) akarjuk áthelyezni. Ezt rekurzív módon, ismét ugyanezt az eljárást alkalmazva végezzük el. Miután befejeztük, az 5. és kisebb lemezek mind a C rúdon(cél) lesznek.


Hívásfa

  A futtatásakor történő események szemléltetésének egy másik módja a hívásfa. Ez az összes hívás grafikus ábrázolása. Az alább látható ábra egy bináris fagráfot tartalmaz, amit a lemezek áthelyezésének könnyebb megértésére hoztunk létre. A START-ból indulva hasonlóan járjuk be a gráfot, mint a mélységi preorder bejárás egészen a STOP-ig, ahol a lemezek áthelyezése már sikerült.

  A hívásfában minden egyes függvényhívást csomópontnak nevezünk. A közvetlenül bármely n csomópont alatt összekapcsolt csomópontok az n függvényhívás által végrehajtott függvényhívásokat jelentik. A teteje alatt például 4(A, B) és 4(B, C) található, mivel ez az a két függvényhívás, amelyet a towerOfHanoi 5(A,C) végrehajt. Alul sok olyan csomópont található, amelyek alatt nincs egyetlen csomópont sem összekapcsolva - ezek az alapeseteket képviselik.
