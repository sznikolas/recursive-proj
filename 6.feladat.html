Összefésülő rendezés - Merge sort


Feladat: A QuickSorthoz hasonlóan a Merge Sort is egy oszd meg és uralkodj algoritmus. A bemeneti tömböt két felére osztja, meghívja magát a két felére, majd egyesíti a két rendezett felet. A merge() függvényt a két fél összevonására használja. A merge(arr, l, m, m, r) egy kulcsfolyamat, amely feltételezi, hogy arr[l..m] és arr[m+1..r] rendezett, és a két rendezett résztömböt egybeolvasztja.



Összefésülő rendezés

N királynő probléma

  Az N királynő-probléma egy sakkfeladvány, lényege a következő: hogyan lehet N királynőt (vezért) úgy elhelyezni egy N×N-es sakktáblán, hogy a sakk szabályai szerint ne üssék egymást. Ehhez a királynő/vezér lépési lehetőségeinek ismeretében az kell, hogy ne legyen két bábu azonos sorban, oszlopban vagy átlóban.

  A nyolckirálynő-probléma egy példa az ennél általánosabb „n királynő problémára”, ami azt a kérdést veti fel, hányféleképpen lehet lerakni n darab királynőt egy n×n-es táblán.

  Az alábbiakban egy 4x4-es táblát találunk, ahol 4 királynőt kell elhelyeznünk a szabályoknak megfelelően bemutatva a rekurziót és a backtracking-et.


Összefésülő rendezés megoldása C nyelvben
                
//C program a összefésülő rendezés megoldására

#include <stdio.h>
#include <stdlib.h>

/*Összefésüli az arr[] két résztömbjét
Az első résztömb: arr[l..m]
A második résztömb: arr[m+1..r]*/
void merge(int arr[], int l, int m, int r)
{
    int i, j, k;
    int n1 = m - l + 1;
    int n2 = r - m;

    //ideiglenes tömbök létrehozása 
    int L[n1], R[n2];

    //Adatok másolása az ideiglenes tömbökbe: L[] és R[]
    for (i = 0; i < n1; i++)
        L[i] = arr[l + i];
    for (j = 0; j < n2; j++)
        R[j] = arr[m + 1 + j];

    //Az ideiglenes tömbök összemergelése az arr[l..r] tömbbel
    i = 0; //Az első résztömb kezdeti indexe
    j = 0; //A második résztömb kezdeti indexe
    k = l; //Az összefésült résztömb kezdeti indexe
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) {
            arr[k] = L[i];
            i++;
        }
        else {
            arr[k] = R[j];
            j++;
        }
        k++;
    }

    //A fennmaradó elemek másolása L[], ha vannak ilyenek.
    while (i < n1) {
        arr[k] = L[i];
        i++;
        k++;
    }

    //A fennmaradó elemek másolása R[], ha vannak ilyenek.
    while (j < n2) {
        arr[k] = R[j];
        j++;
        k++;
    }
}

/*l a bal oldali index és r a rendezni kívánt
    arr résztömb jobb oldali indexe*/
void mergeSort(int arr[], int l, int r)
{
    if (l < r) {
        /*Ugyanaz, mint (l+r)/2, de nagy l és r esetén 
            elkerülhető a túlcsordulás*/
        int m = l + (r - l) / 2;

        //Rendezi az első és második felét
        mergeSort(arr, l, m);
        mergeSort(arr, m + 1, r);

        merge(arr, l, m, r);
    }
}

/*Haszonossági függvények*/
/*függvény tömb kiíratására */
void printArray(int A[], int size)
{
    int i;
    for (i = 0; i < size; i++)
        printf("%d ", A[i]);
    printf("\n");
}

/*fő kód*/
int main()
{
    int arr[] = { 12, 11, 13, 5, 6, 7 };
    int arr_size = sizeof(arr) / sizeof(arr[0]);

    printf("A megadott tömb: \n");
    printArray(arr, arr_size);

    mergeSort(arr, 0, arr_size - 1);

    printf("\nA rendezett tömb: \n");
    printArray(arr, arr_size);
    return 0;
}


Forrás
  • 1. Keressük meg a középső pontot, hogy a tömböt két részre osszuk:
    middle m = l+ (r-l)/2
    2. Hívjuk a mergeSort programot az első felére:
    call mergeSort(arr, l, m)
    3. Hívjuk a mergeSortot a második felére:
    call mergeSort(arr, m+1, r)
    4. Egyesítse a 2. és 3. lépésben rendezett két felet:
    call merge(arr, l, m, r)


Az algoritmus leírása

MergeSort(arr[], l, r)

Ha r > l

1. Keressük meg a középső pontot, hogy a tömböt két részre osszuk:
middle m = l+ (r-l)/2

2. Hívjuk a mergeSort programot az első felére:
call mergeSort(arr, l, m)

3. Hívjuk a mergeSortot a második felére:
call mergeSort(arr, m+1, r)

4. Egyesítse a 2. és 3. lépésben rendezett két felet:
call merge(arr, l, m, r)

A tömböt rekurzívan két részre osztjuk addig, amíg a mérete 1 nem lesz. Amint a méret 1 lesz, az egyesítési folyamatok működésbe lépnek, és elkezdik a tömbök összeolvasztását, amíg a teljes tömb össze nem áll.
